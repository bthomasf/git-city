## ADDED Requirements

### Requirement: 城市视图准实时刷新
系统 MUST 支持在不刷新浏览器页面的情况下，定期或在检测到新数据时自动刷新城市视图，使团队与个人城市在合理延迟范围内反映最新的 GitLab 活动。

#### Scenario: 定时刷新团队城市
- **WHEN** 用户保持团队城市页面处于打开状态
- **THEN** 系统 SHALL 以配置的时间间隔（例如 1 分钟或 5 分钟）轮询城市数据 API
- **THEN** 如果存在新的聚合数据，前端 SHALL 使用动画或平滑过渡的方式更新城市楼宇高度与状态
- **THEN** 如果在某个周期内无新数据，系统 SHALL 不强制重绘城市以避免不必要的闪烁

### Requirement: 刷新对性能与体验的约束
系统 MUST 在刷新策略上平衡数据新鲜度与系统资源消耗，避免过于频繁的 API 调用和大规模重绘对内网 GitLab 或前端性能造成明显影响。

#### Scenario: 刷新频率配置与限制
- **WHEN** 管理者通过配置将刷新间隔设置为过小值
- **THEN** 系统 SHOULD 对刷新间隔施加合理的下限（例如不小于 30 秒），以防止对 GitLab 造成过大压力
- **THEN** 当刷新失败或超时时，系统 SHALL 在 UI 上以中文提示用户当前数据可能不是最新状态
